Memory pipeline & comparator logic: At reinforcement time T, the accumulator value n(T) is encoded into working memory with multiplicative noise σwm → n*. On each new trial, reference memory K* is sampled from n* with retrieval noise σrm. Crucially, n* is stored at T, not at response time — so K* does not drift across trials. The comparator continuously computes a similarity score = 1 − |n − K*| / K*, rising from 0 toward 1.0 as the accumulator approaches K*. When similarity ≥ 1 − b (threshold b), the comparator declares MATCH and triggers a response. The amber band on CH1 shows the response window K* ± b·K*. Set η noise parameters > 0 and run 30+ trials to observe the scalar property: CV stays roughly constant across different values of T.
